Router# show ip bgp summary
Secure BGP Internet Edge & DMZ Architecture

The goal: keep the network's heartbeat to the internet alive even if a provider fails, while exposing exactly one server to the outside world — and never opening a door into anything else.

  • Dual-homed internet gateway peering via eBGP with multiple upstream ISPs (AS 15169) under a local AS 65000 — if one provider link drops, traffic reroutes through the other automatically, with no manual intervention.
  • Dynamic route redistribution between the eBGP edge and a Multi-Area OSPF Area 0 backbone, backed by floating static routes (Administrative Distance 10) that only kick in the instant the primary path disappears.
  • PAT overload lets the entire internal LAN share one public IP, while a separate static NAT rule punches a single, tightly scoped hole — public traffic on TCP/80 only — straight through to the DMZ web server on VLAN 200.
  • A zone-based Extended ACL ("SECURE-DMZ") uses TCP flag matching (established) to allow legitimate return traffic back in, while refusing anything that looks like an attempt to hop from the DMZ into the trusted internal subnets.

Router# show spanning-tree summary
Redundant Enterprise Campus Network Architecture

A campus backbone built the way real enterprises design their core: no single switch, link, or path is ever allowed to take the whole network down with it.

  • A high-availability "Triangle" Core built from three Cisco 3650 Layer 3 switches, fully meshed so any one switch or link can fail without anyone losing connectivity.
  • STP priorities tuned by hand (4096 / 8192 / 12288) so the topology always knows exactly which switch is in charge, paired with LACP (802.3ad) EtherChannels that bundle multiple physical links into one fast, resilient logical path.
  • Inter-VLAN routing through SVIs with DHCP relay (ip helper-address) keeps Student (VLAN 10), Staff (VLAN 20), and Management (VLAN 99) traffic cleanly separated, while every device still gets an IP address automatically.
  • BPDU Guard and PortFast on access ports stop accidental loops before they start; Root Guard on the core downlinks blocks any rogue switch from hijacking the spanning-tree topology.
  • A routed point-to-point /30 link (no switchport) ties the core directly into a Cisco ISR4321 router, giving the whole campus verified, end-to-end reachability out to the WAN.
Triangle Core — zero single points of failure
Router# show ip route static
Enterprise Multi-Site Network Architecture

Built to mirror how a growing company connects new branch offices back to one central site without re-cabling everything — a single physical link doing the work of several.

  • A hub-and-spoke WAN where 802.1Q encapsulation (router-on-a-stick) lets one trunk link carry traffic for multiple departments at once, with the router sorting and routing between them.
  • IP addressing sized exactly to need using VLSM — /28 for each branch LAN, /30 for the point-to-point links between sites — instead of wasting address space.
  • Static routing ties every site back together end-to-end, including a verified path out to public DNS (8.8.8.8).
